# Agent 3Bd V2 - ملف تعريف المشروع

## نظرة عامة على المشروع

تطبيق جوال متقدم لإدارة الوكلاء الذكية مع واجهة سيبرنية حديثة، يدعم التفاعل مع نماذج ذكاء اصطناعي متعددة ويوفر أدوات متقدمة لإدارة المهام والذاكرة.

### المميزات الرئيسية
- دعم متعدد المزودين: DeepSeek, Groq, OpenAI, Together
- إدارة الوكلاء: إنشاء وتخصيص وكلاء ذكية متعددة
- محادثات ذكية مع واجهة حديثة وسهلة الاستخدام
- نظام شامل لإدارة الأذونات
- التخزين المحلي للمحادثات والبيانات
- واجهة سيبرنية حديثة مع ألوان ذهبية وسوداء

### الأذونات المدعومة
- الكاميرا والميكروفون
- الموقع الجغرافي وجهات الاتصال
- الوسائط وأجهزة استشعار الجسم
- الأجهزة المجاورة والرسائل النصية

## المتطلبات الوظيفية

### الشاشات الرئيسية

#### 1. الشاشة الرئيسية (Home Screen)
- عرض الوكلاء النشطين والمحادثات الأخيرة
- شريط بحث سريع
- زر إضافة وكيل جديد

#### 2. شاشة المحادثة (Chat Screen)
- سجل المحادثة الكامل
- حقل إدخال مع أيقونات الإجراءات
- عرض حالة الوكيل (يكتب، جاهز، خطأ)
- أيقونات الأدوات المتاحة

#### 3. شاشة الإعدادات (Settings Screen)
- مفاتيح API للمزودين المختلفين
- اختيار النموذج الافتراضي
- إعدادات درجة الحرارة والرموز القصوى
- إدارة الأذونات

#### 4. إدارة الوكلاء (Agent Management)
- قائمة الوكلاء المتاحة
- تفاصيل كل وكيل
- إعدادات الوكيل المخصصة

#### 5. شاشة الذاكرة (Memory Screen)
- البيانات المحفوظة والسياق
- سجل الإجراءات المنفذة
- الملفات والمرفقات المحفوظة

#### 6. شاشة الأدوات (Tools Screen)
- قائمة الأدوات المتاحة
- وصف كل أداة
- حالة الأداة

### التدفقات الرئيسية

#### تدفق المحادثة الجديدة
1. الضغط على "محادثة جديدة"
2. اختيار الوكيل المطلوب
3. الانتقال إلى شاشة المحادثة
4. كتابة الرسالة الأولى
5. إرسال الرسالة وعرض الرد

#### تدفق تكوين API
1. الانتقال إلى شاشة الإعدادات
2. اختيار المزود
3. إدخال مفتاح API
4. اختبار الاتصال
5. حفظ الإعدادات

### متطلبات التصميم

#### الألوان
- خلفية سوداء داكنة (#0c0e10)
- خلفية ثانوية رمادية داكنة (#292929)
- خلفية حقول رمادية متوسطة (#393939)
- لون أساسي ذهبي (#c9b974)
- نص أبيض (#ffffff)
- نص ثانوي رمادي فاتح (#a0a0a0)
- لون خطأ أحمر (#e76a5e)
- لون نجاح أخضر (#4ade80)

#### التوجيه والتصميم
- التوجيه العمودي (Portrait) فقط
- تصميم مناسب للاستخدام بيد واحدة
- حد أدنى للعناصر التفاعلية 44x44 بكسل
- خطوط: Outfit للنصوص العادية، IBM Plex Mono للأكواد
- رسوم متحركة سلسة (250-300ms)
- وضع ليلي كامل الدعم

## الفئة المستهدفة
- مطورون ومهندسون ذكاء اصطناعي
- مستخدمون متقدمون لتطبيقات الذكاء الاصطناعي
- فرق العمل التي تحتاج لوكلاء ذكية متعددة
- مطورون تطبيقات الجوال

## الأهداف
1. توفير واجهة مستخدم حديثة وسهلة الاستخدام
2. تمكين إدارة متعددة الوكلاء من منصة واحدة
3. تسهيل التفاعل مع نماذج ذكاء اصطناعي متعددة
4. توفير نظام شامل لإدارة الأذونات والأدوات
